Вспомним HTML
HTML
  • HyperText Markup Language (язык разметки гипертекста) - тэговый язык разметки документов
  • термин, введённый Тедом Нельсоном в 1965 году для обозначения «текста ветвящегося или выполняющего действия по запросу»
  • текст, сформированный с помощью языка разметки, содержащий в себе гиперссылки
  • для передачи гипертекстовых документов применяется протокол HTTP (протокол передачи гипертекста) версии 1.1 (RFC 2616).
    •

    Гиперссылка
  • часть гипертекстового документа, ссылающаяся на другой элемент (команда, текст, заголовок, примечание, изображение) в самом документе, на другой объект (файл, каталог, приложение), расположенный на локальном диске или в компьютерной сети, либо на элементы этого объекта.
  • служит для связи документов
    •
    Сервер
  • программное обеспечение
  • обслуживает запросы от клиентов
    •

    Клиент
  • программное обеспечение. (Например браузер)
  • выполняет запросы к серверу
  • отображает ответы сервера пользователю
    •
    Взаимодействие клиента с сервером
  • поиск IP адреса сервера при помощи системы доменных имен (DNS)
  • установка соединения с сервером (стандартно на 80 порт)
  • HTTP запрос (request) клиента к серверу
  • HTTP ответ (response) сервера клиенту
    •
    HTTP запрос
  • состоит из заголовка запроса и тела запроса, разделенные пустой строкой. Тело запроса может отсутствовать.
  • методы (иначе говоря, команда HTTP)
  • параметры
    •

    Методы запроса
  • GET - запрос документа
  • HEAD - запрос заголовка документа
  • POST - этот метод применяется для передачи данных. Сами данные следуют в последующих строках запроса в виде параметров
  • PUT - разместить документ на сервере (используется редко). Запрос с этим методом имеет тело, в котором передается сам документ.
  • Ресурс - это путь к определенному файлу на сервере, который клиент хочет получить
  • Версия протокола - версия протокола HTTP, с которой работает клиентская программа.
    •
    HTTP ответ
  • состоит из заголовка и тела, разделенные пустой строкой.
  • версия протокола - аналогичен соответствующему параметру запроса.
  • код ошибки - кодовое обозначение "успешности" выполнения запроса. Код 200 означает "все нормально" (OK).
  • словесное описание ошибки - "расшифровка" предыдущего кода. Например для 200 это OK, для 500 - Internal Server Error.
  • параметры

    Структура HTML-документа
  • Любой документ на языке HTML представляет собой набор элементов, причём начало и конец каждого элемента обозначается специальными пометками — тегами.
  • элементы могут быть пустыми
  • элементы могут иметь атрибуты, определяющие какие-либо их свойства
  • регистр, в котором набрано имя элемента и имена атрибутов, в HTML значения не имеет

    • <strong>Текст между — открывающим и закрывающим.</strong>
<a href="http://www.example.com">элемент содержит атрибут href.</a>
А вот пример пустого элемента: <br>
  • элементы могут быть вложенными
    • <b>Этот текст будет полужирным,
<i>а этот - ещё и курсивным</i>
</b>
    Примеры
  • Корневой тэг <html>...</html> - говорит браузеру, что содержит HTML
  • <head>...</head> - содержит информацию о документе
  • <body>...</body> - отображаемое содержимое документа
